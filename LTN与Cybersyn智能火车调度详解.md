# LTN 与 Cybersyn — 智能火车调度系统详解

> 整理时间：2026年6月  
> 兼容版本：Factorio 1.1 ~ 2.0 + Space Age DLC

---

## 为什么需要它们？

原版火车只能按固定时刻表跑固定路线。当你有 50 个矿站、30 个工厂站、100 列火车时，手动调度是噩梦。这两个 Mod 把火车变成「物流机器人」——自动按供需匹配、自动派车、自动回库。

```
原版物流网络              智能火车网络
─────────────────     ─────────────────
请求箱 (Requester)  →  请求站 (Requester)
供应箱 (Provider)   →  供应站 (Provider)
机器港 (Roboport)   →  车库站 (Depot)
物流机器人           →  火车
```

## 📖 LTN — Logistic Train Network（物流火车网络）

| 项目 | 详情 |
|------|------|
| **作者** | Optera（2.0 版由 hgschmie 维护） |
| **下载量** | **28.1 万** |
| **年龄** | 9 年（老祖宗级别） |
| **兼容** | 0.14 ~ 2.0 + Space Age |
| **版本** | 2.8.0（持续更新中） |
| **许可** | 私人使用，禁止修改 |
| **Mod 站** | https://mods.factorio.com/mod/LogisticTrainNetwork |

### 三种车站

| 车站类型 | 灯色 | 作用 |
|----------|------|------|
| 🏠 **Depot（车库）** | 黄色 | 火车在此待命，等待调度。任务完成后返回这里 |
| 📤 **Provider（供应站）** | 绿色 | 矿井、工厂产出端——"我有这些东西可以提供" |
| 📥 **Requester（请求站）** | 蓝色 | 工厂输入端——"我需要这些东西" |

### 工作流程

```
1. 请求站发出信号："我需要 8000 铁板"
         │
2. LTN 中央调度器扫描所有供应站
         │
3. 找到矿站 A："我有 24000 铁板，够装"
         │
4. 从最近的车库调一列空闲火车
         │
5. 自动生成时刻表：车库 → 矿站A（装货）→ 请求站（卸货）→ 车库
         │
6. 火车出发 ✅  卸完货后自动回库待命
```

### 电路信号

LTN 通过电路信号控制车站行为：

| 信号 | 含义 |
|------|------|
| **物品信号（正数）** | 本站可提供此物品的数量 |
| **物品信号（负数）** | 本站请求此物品的数量 |
| **请求阈值** | 缺多少才触发一次运输（防止为 5 个铁板派一列火车） |
| **网络 ID** | 隔离网络。不同 ID 的车站互不可见 |
| **优先级** | 数字越大越优先被服务 |

### 关键特性

| 特性 | 说明 |
|------|------|
| 🧩 **网络隔离** | 通过网络 ID 创建多个互不干扰的铁路网 |
| 🔢 **品质支持** | 2.1.0 起支持 Factorio 2.0 品质系统 |
| ⛽ **燃油站** | 2.3.0 新增，火车燃料不足自动绕道加油 |
| 🌍 **跨星球** | 配合 Space Exploration，支持太空电梯跨表面运输 |
| 🛑 **列车限制** | 自动管理车站可停火车数，防止堵塞 |

## 🆕 Cybersyn — Project Cybersyn（赛博协同）

| 项目 | 详情 |
|------|------|
| **作者** | mami |
| **下载量** | **5.1 万** |
| **年龄** | 较新 |
| **兼容** | 1.1 ~ 2.0 + Space Age |
| **版本** | 2.0.48（活跃更新中） |
| **许可** | 开源 |
| **Mod 站** | https://mods.factorio.com/mod/cybersyn |

### 命名来源

取自智利 1970 年代的真实项目——[**赛博协同控制工程（Project Cybersyn）**](https://zh.wikipedia.org/wiki/赛博协同控制工程)，萨尔瓦多·阿连德政府试图用计算机网络管理全国经济。Factorio 里你也是用网络调度整个工业帝国——名字非常应景。

### 唯一新增实体——赛博组合器（Cybernetic Combinator）

放在车站旁边，有 **5 种模式**：

| 模式 | 作用 |
|------|------|
| 📡 **Station** | 正信号 = 提供物品，负信号 = 请求物品 |
| 🏠 **Depot** | 停在这里的火车加入调度网络 |
| ⛽ **Refueler** | 燃油站，火车燃料不够时自动绕道 |
| 🎛 **Station Control** | 可选的第二组合器，设置每个物品的阈值和优先级 |
| 🚃 **Wagon Control** | 控制每个车厢精确装载/卸载什么（万能车站的核心） |

### 只需 3 个信号

| 信号 | 作用 |
|------|------|
| **物品信号** | 正数 = 供应，负数 = 请求 |
| **请求阈值** | 同时充当供应和请求的触发门槛 |
| **网络 ID**（默认信号 A） | 隔离不同网络 |

> 相比之下，LTN 需要更多组合器和信号配置。Cybersyn 的哲学是「少即是多」。

### 关键特性

| 特性 | 说明 |
|------|------|
| ⚡ **高性能** | 自定义中央规划算法，约 3 倍于 LTN 的速度 |
| 🔧 **万能车站** | Wagon Control 模式原生支持——一个车站处理所有物品类型 |
| 🛡 **自动防护** | 自动检测相邻机械臂类型，阻止流体车厢被派去装铁板 |
| 🔌 **跨星球** | 自动检测 Space Exploration 太空电梯 |
| 📊 **运送输出** | Station Control 组合器输出进行中的运送信息，避免重复下单 |

## 🆚 LTN vs Cybersyn 详细对比

| | LTN | Cybersyn |
|------|------|------|
| ⚡ **性能** | 基准线 | **约 3 倍更快**（自定义中央规划算法） |
| 🧩 **复杂度** | 需要多个组合器配置 | 一个赛博组合器搞定基础配置 |
| 📖 **学习曲线** | 陡峭（信号逻辑复杂） | 稍缓（只需 3 个信号） |
| 📚 **文档/教程** | 丰富（9 年积累） | 较少（但有官方蓝图示例） |
| 👥 **用户量** | 28.1 万 | 5.1 万 |
| 🔄 **维护** | 持续（由社区维护） | 活跃（原作者持续更新） |
| 🚗 **万能车站** | 可以实现但配置复杂 | ✅ 原生支持（Wagon Control 模式） |
| 🛡 **误操作保护** | 一般 | ✅ 自动阻止不兼容的火车类型 |
| ⛽ **加油方式** | 燃油站（燃油信号 + 阈值配置） | Refueler 模式（独立车站类型） |
| 🔌 **跨星球** | 需要子 Mod（LTN-Space Exploration） | 原生自动检测 |
| 🧩 **配套生态** | 大（LTN Combinator、LTN Tracker 等） | 小 |

## 🎯 选哪个？

| 你的情况 | 推荐 |
|----------|------|
| 第一次玩火车调度 Mod | **Cybersyn** — 更简单、更现代、性能更好 |
| 喜欢看教程/视频学习 | **LTN** — 9 年沉淀，网上教程一搜一大把 |
| 工厂已经很大（megabase） | **Cybersyn** — 3 倍性能优势在千车级工厂里很关键 |
| 玩 Space Exploration | 两者都行，Cybersyn 原生检测太空电梯更方便 |
| 不想折腾信号 | **Cybersyn** — 只需 3 个信号 |
| 需要成熟配套生态 | **LTN** — 配套子 Mod 多 |
| 手头已有 LTN 蓝图 | **LTN** — 不用重新学、重新造轮子 |

## 🚀 实际使用示例（以 Cybersyn 为例）

### 铁板供应站（矿井）

```
┌─────────────────────────────────┐
│                                 │
│  矿机 → 箱子（缓存）→ 赛博组合器    │
│                     │           │
│                     ▼           │
│              Station 模式        │
│              铁板信号 = +24000   │
│              (我有 24000 铁板)    │
│                                 │
│  车站名：Iron_Load               │
│  灯色：🟢 绿色（供应中）           │
└─────────────────────────────────┘
```

### 铁板请求站（炼钢厂）

```
┌─────────────────────────────────┐
│                                 │
│  恒准组合器 → 铁板 = -16000      │
│  请求阈值 = 8000（少于 8000 别派车)│
│              │                  │
│              ▼                  │
│         赛博组合器               │
│         Station 模式             │
│              │                  │
│              ▼                  │
│  车站名：Iron_Unload              │
│  灯色：🔵 蓝色（请求中）           │
└─────────────────────────────────┘
```

### 车库

```
┌─────────────────────────────────┐
│                                 │
│  车站 + 赛博组合器（Depot 模式）    │
│  停 3 列 1-4 火车（车头+4 货厢）   │
│  灯色：🟡 黄色（待命中）           │
│                                 │
└─────────────────────────────────┘
```

### 调度结果

```
矿站铁板 > 8000 → Cybersyn 检测到
炼钢厂缺 > 8000 → Cybersyn 检测到
         │
         ▼
调度器匹配 → 从最近车库派一列空闲火车
         │
         ▼
自动时刻表：车库 → Iron_Load（装满）→ Iron_Unload（卸空）→ 回库
```

## ⚠️ 重要注意事项

| 注意 | 说明 |
|------|------|
| ⚠️ **只能装一个** | LTN 和 Cybersyn 互斥，功能完全重叠，选一个装 |
| 🚇 **2.0 中断系统** | Factorio 2.0 的火车中断（Interrupts）已实现部分智能调度，小工厂可能够用 |
| 📦 **缓存空间** | 请求站必须有能力接收**至少 2 列满载火车**的量，否则会堵死 |
| 🛤 **铁路设计** | 智能调度对铁路布局要求更高——必须有足够候车区和回路 |
| 🔢 **网络 ID** | 确保所有车站使用相同的网络 ID 信号，否则调度器看不到 |
| 🚦 **信号灯** | 智能火车多了以后，铁路信号灯必须正确布置，死锁风险上升 |

## 📦 快速上手检查清单

### 通用步骤

- [ ] 安装 Mod（LTN 或 Cybersyn，二选一）
- [ ] 建一个 Depot（车库）站，放赛博组合器设为 Depot 模式
- [ ] 停一列火车在 Depot，设为自动模式
- [ ] 建 Provider（供应）站，用信号声明可提供的物品
- [ ] 建 Requester（请求）站，用信号声明需要的物品
- [ ] 确保三个站属于同一网络 ID
- [ ] 等待火车自动调度 🎉

### LTN 额外步骤

- [ ] 用 LTN Combinator 辅助信号配置（可选但推荐）
- [ ] 设置默认网络 ID（Mod 设置中，2.2.0 后默认为 0）

### Cybersyn 额外步骤

- [ ] 下载官方示例蓝图（GitHub 上有导入码）
- [ ] 设置请求阈值（必须 ≤ 一列火车的运力）
